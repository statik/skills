1. Design Thesis

Every plan deserves a format that reviewers can scan, navigate, and share without reading a chat transcript. The design-memo skill makes self-contained HTML design memos the default output for any non-trivial plan, architecture proposal, or decision record produced by Claude.

The skill asks before generating HTML for simple plans — it is a default, not a mandate.

The visual language is Tufte-inspired editorial layout with a cinematic Arrival palette: foggy slate backgrounds, charcoal ink, muted olive and amber structure, and blue-gray accents for technical emphasis. Lightweight scrollytelling (progress bar, section rail, chapter indicator) gives the document presentation-like affordances while remaining a readable long-form memo.

This document is itself produced by the skill — a proof of concept for reviewers to evaluate the format alongside the integration changes.

3. Key Design Decisions

Trigger scope: default with an escape hatch

The skill activates for any plan or design request. For simple plans (single-step fixes, quick answers), it asks the user whether they prefer plain text. This avoids both extremes: ignoring the skill for small tasks and forcing a full HTML document for a two-line fix.

Output location: docs/ preferred

HTML memos are placed in a docs/ subdirectory if one exists. If not, the skill asks before creating one, falling back to the current directory. This keeps project roots clean without imposing structure on repos that don't have a docs convention.

CSS variable renames

The original draft used --red for an olive green (#4a4f3f) and --rose for an amber (#9a7a55). These names now match their actual colors: --olive and --amber. Hex values are unchanged.

Semantic variable names prevent mistakes when extending the palette or building conditional styles.
Semantic variable names prevent mistakes when extending the palette or building conditional styles.

Palette contrast fix

--muted was darkened from #65706f to #4a5250 to meet WCAG 2.1 AA (4.5:1 against --paper: #cfd4cf). The skill also notes that --muted is for secondary text only — essential content must use --ink.

plugin.json: description only

The name field in plugin.json was left as dns-troubleshooter to avoid breaking existing installations. Only the description was broadened to acknowledge the collection now has multiple skills.

4. Skill Structure

The SKILL.md follows the Agent Skills Specification and mirrors the patterns established by dns-troubleshooter. Key sections: 

skills/design-memo/
  SKILL.md          # YAML frontmatter + 13 instruction sections

The instruction sections, in order:

  1. Purpose — when and why to produce an HTML memo
  2. Output — self-contained, inline CSS/JS only
  3. Output Location — docs/ preferred, ask if missing
  4. Output File Naming — <topic>-design-memo.html
  5. Visual Direction — Tufte layout, cinematic palette
  6. Default Palette Variables — 14 CSS custom properties
  7. Scrollytelling Interaction — progress bar, rail, chapter indicator
  8. Accessibility — WCAG AA, ARIA, keyboard nav, reduced motion
  9. Content Structure — 7-part technical design template
  10. Code Listings — inline highlighting, draft labels
  11. Review Quality Bar — caveats surfaced, test cases included
  12. HTML Implementation Notes — semantic sections, CSS properties, no-JS fallback
  13. HTML Skeleton — complete starter template (~200 lines)
No evals yet. HTML presentation quality is hard to score automatically. A future enhancement could use a headless browser to verify structural correctness (sections present, CSS variables defined, no broken JS), but that is out of scope for the initial integration.

5. Accessibility Provisions

Requirement Implementation
Color contrast All text meets WCAG 2.1 AA (4.5:1 body, 3:1 large). --muted darkened to comply.
Keyboard navigation Rail dots are <button> elements with :focus-visible outlines.
Screen readers role="navigation" on rail, role="progressbar" with aria-value* on progress, aria-current="step" on active dot, aria-live="polite" on chapter indicator.
Reduced motion prefers-reduced-motion media query disables transitions.
No-JS fallback All content visible in document order. JS adds interactivity only.

7. Confidence & Caveats

  • High confidence: The skill validates, follows all existing patterns, and the HTML skeleton produces a working document (this memo is evidence).
  • Medium confidence: The trigger scope ("default for plans, ask for simple ones") requires judgment from the agent. Edge cases will surface with use — the phrasing may need tuning.
  • Known gap: No automated evals. Quality depends on manual review until a structural eval is built.
  • Known gap: release.yml only handles dns-troubleshooter. A future PR should make it multi-skill-aware.
  • Palette note: The --muted color was adjusted for contrast. If the original lighter shade is preferred for decorative elements, a separate --muted-light variable could be added later.
